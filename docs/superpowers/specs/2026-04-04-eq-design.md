# Feature 2 — 10-Band Parametric EQ: Design Spec

**Date:** 2026-04-04
**Status:** Approved

---

## Summary

A 10-band parametric EQ implemented with Web Audio `BiquadFilterNode`s. Opens as a bottom drawer above the PlayerBar (`E` key or EQ button). The entire panel is a draggable SVG frequency response curve — no separate sliders. Presets load from a constant and persist custom saves to Dexie.

---

## Decisions Made

| Question | Decision |
|----------|----------|
| Panel placement | Bottom drawer — slides up above PlayerBar |
| Curve style | Full SVG bezier from `getFrequencyResponse()`, 512 points, filled |
| Interaction | Drag dots directly on the curve (up/down = gain ±15 dB) |
| State location | Local `useReducer` in `EQPanel.tsx` — not in global playerContext |
| Bypass impl | Zero all BiquadFilter gains (no disconnection — avoids audio clicks) |

---

## Audio Graph

**Current chain:**
```
src → analyser → gainNode → destination
```

**New chain:**
```
src → analyser → eq[0] → eq[1] → … → eq[9] → gainNode → destination
```

All 10 `BiquadFilterNode`s are created in `ensureAudio()` in `playerContext.tsx` and stored as a module-level array `eqNodes: BiquadFilterNode[]`. They are connected once on creation and never disconnected. Bypass is implemented by setting all band gains to 0 (originals saved in `eqBypassGains` array for restoration).

`playerContext.tsx` exposes two new callbacks:
- `setEQBandGain(bandIndex: number, gainDb: number): void` — updates `eqNodes[i].gain.value`
- `setEQBypass(on: boolean): void` — zeroes / restores all gains

These are the only surface area. The EQ panel owns all state; it calls these to write to the audio graph.

## EQ State (`EQPanel.tsx`)

```ts
interface Band {
  freq: number;
  gain: number;   // -15 to +15 dB
  q: number;
  type: BiquadFilterType;
}

interface EQState {
  bands: Band[];
  bypass: boolean;
  presetName: string;  // '' = unsaved custom
}

type EQAction =
  | { type: 'SET_BAND_GAIN'; index: number; gain: number }
  | { type: 'TOGGLE_BYPASS' }
  | { type: 'LOAD_PRESET'; bands: Band[]; name: string }
  | { type: 'RESET_FLAT' }
  | { type: 'SET_PRESET_NAME'; name: string };
```

`INITIAL_BANDS` is the 10-band array above with all gains at 0. The reducer is pure — easy to unit test.

### Header row

- "EQ" label (text-white/60 text-xs font-semibold tracking-widest uppercase)
- Bypass toggle button — when active: text-white, ring; inactive: text-white/30
- Preset chips (horizontal scroll on overflow): 8 built-in + custom saved. Active preset highlighted (bg-white/10 border-white/20)
- "Save" button — opens inline name input → on Enter: writes to Dexie, adds to custom list
- Close `✕` button (or press `E` again)

### Curve area (`EQCurve.tsx`)

SVG element, `width="100%"`, fixed `viewBox="0 0 800 130"`.

**Grid lines:** horizontal dashed lines at +15, +6, 0, -6, -15 dB. Vertical lines at decade boundaries (20, 200, 2k, 20k Hz). Labels in `text-white/20`.

**Frequency response path:**
1. Build an array of 512 log-spaced frequencies: `f[i] = 20 * (1000)^(i/511)` (20 Hz → 20 kHz)
2. For each frequency, compute each band's dB response **mathematically** from `{ freq, gain, q, type }` using the standard BiquadFilter transfer function (no audio node access needed — works in tests and SSR). Helper: `biquadResponse(band, f) → dB`
3. Sum all 10 bands' dB contributions at each frequency point
4. Map `(freq, dBSum)` → SVG `(x, y)` using log-scale x, linear y (±15 dB range)
5. Render as `<path d="M x0,y0 L x1,y1 ..." />` — polyline is sufficient; bezier adds complexity without visible benefit at 512 points
6. Filled area: `<path>` closed to the 0 dB baseline, `fill="rgba(124,58,237,0.15)"`, stroke `#7c3aed`

This recomputes on every `EQState` change via `useMemo`. The math is ~20 lines — no external library needed.

**Band dots:**
- 10 `<circle>` elements, each positioned at `(logFreqToX(band.freq), gainToY(band.gain))`
- `r=6`, `fill="#7c3aed"`, `stroke="#000"` stroke-width 2
- `cursor: ns-resize`
- On `pointerdown` → `setPointerCapture` → track vertical drag → `dispatch SET_BAND_GAIN(index, clamp(newGain, -15, 15))`
- On drag: also calls `setEQBandGain(index, gain)` directly (live audio update, no batching delay)
- Active dot (being dragged): `r=8`, tooltip `<rect>+<text>` showing `+4.5 dB`
- Double-click resets band to 0 dB

**Bypass state:** When `bypass=true`, curve opacity drops to 0.3 and dots become non-interactive.

## Explicit Non-Scope (V1)

- Per-band frequency adjustment (fixed frequencies in V1)
- Per-band Q adjustment (fixed Q per type in V1)
- High-pass / low-pass filters at chain boundaries
- Band type switching (peaking ↔ notch)
- Pitch-shift mode
- Real-time spectrum analyser overlay on curve

These are all valid future additions — none are needed for a fully functional, professional-quality EQ.

---

## Browser Support

All features use `BiquadFilterNode` and `getFrequencyResponse()` — both universally supported across Chrome, Firefox, Safari, Edge, iOS Safari, and Android Chrome.
