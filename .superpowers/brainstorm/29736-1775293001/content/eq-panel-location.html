EQ Panel Placement

UX recommendation with rationale — click to select your preference

Recommended

A — Bottom Drawer ★ Expert pick

Slides up above the PlayerBar. EQ stays visible while music plays — you tweak and hear changes immediately. This is how Spotify, foobar2000, and Vinyls handle EQ. Doesn't disrupt the visualizer. Naturally anchored to the audio controls beneath it.

Pros

  • Continuous visibility while listening
  • Physically close to the controls it affects
  • Easy to dismiss (drag down or press E)
  • Works perfectly on all screen sizes

Cons

  • Partially covers the visualizer (intentional — still visible)

B — Right Sidebar

Slides in from the right, like the Queue sidebar. Consistent mental model — sidebars are panels. But EQ needs width for 10 band sliders + frequency curve. Sidebars are narrow by convention, causing a cramped layout.

Pros

  • Consistent with QueueSidebar pattern
  • Non-destructive to bottom area

Cons

  • EQ needs width — sidebars are narrow
  • Far from the PlayerBar it relates to
  • Covers half the visualizer

C — Modal Overlay

Centered dialog over everything. Standard for settings dialogs — but EQ is a live tool, not a settings screen. Dimming the background creates a false sense of "stopping" to configure, when really EQ is something you tweak while listening.

Pros

  • Full width available for the curve
  • Familiar pattern for settings

Cons

  • Dims visualizer — breaks immersion
  • Feels like a settings dialog, not a live tool
  • Harder to dismiss accidentally
